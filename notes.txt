Neste momento, passamos a implementar o padrão repository

Lembrando que o padrão repository sugere a criação de um repository por Aggregate.

Aggregate é um conjunto de objetos de domínio que podem ser tratados como uma única unidade. Por exemplo:
Só faz sentido haver uma itens de um pedido se houver um pedido. Portanto, o pedido seria o Aggregate Root

Criamos uma interface para nosso repositorio dentro do pacote domain.model.repository, pois a interface
não terá nenhum conhecimento de infraestrutura, apenas do dominio do negócio

Criamos uma classe de repositório dentro do pacote infrastructure.repository, pois esta classe terá conhecimento
de infraestrutura
